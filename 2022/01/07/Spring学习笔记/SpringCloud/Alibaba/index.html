Nacos

安装
windows 版本

  1. 需要环境Java8+Maven

  2. 下载Nacos（https://github.com/alibaba/nacos/releases/download/2.0.3/nacos-server-2.0.3.zip）

  3. 运行Nacos （注意：需要对配置文件先进行修改）

  4. 访问 http://localhost:8848/nacos 默认账号密码为nacos

注意：需要对数据库先进行数据库中表的创建

docker 版本

  1. git clone https://github.com/nacos-group/nacos-docker.git cd nacos-docker

  2. docker-compose -f example/standalone-mysql-5.7.yaml up

    注意：刚开始的运行的时候不会离开创建数据库中的表，因此过了一段时间之后可以发现表已经创建完成了，重启nacos即可

流控规则

  • 每秒的请求数控制

    图中显示的是每秒的请求数不能超过1个/s，超过则报错

  • 线程数

    线程数指的是内部处理的线程设定值

  • 关联流控模式

    程序A的并发量达到一定的程度，会限制B的并发量

    比如说购物接口并发量很大导致支付接口压力很大，那么支付接口达到一定的压力之后就会限制购物接口的并发量

  • warmup配置

  • 排队等待配置

    匀速排队，让请求以均匀的速度通过，阈值类型必须设置为QPS

    每秒钟处理一个请求，如果请求太多并且等待时间超过10s，那么就会报错，否则会等待处理

降级

  • RT（响应时间）

    含义：当1s内有>=5个请求过来并且其中有80%的请求的响应时间都>200ms,那么会触发熔断器持续1s，1s之会恢复（不会像豪猪哥那样half恢复）

  • 异常比例

    含义：当1s内有>=5个请求过来并且其中有80%的请求都失败,那么会触发熔断器持续1s，1s之会慢慢恢复

  • 异常数

    含义：当1min内有>=5个请求过来并且其中有100个请求都失败,那么会触发熔断器持续1s，1s之会慢慢恢复（此处必须是都是分钟级别的）

热点key

controller配置

含义：如果1s内testHotKey这个资源中的第1个参数的次数超过1，就会触发熔断，执行deal_testHotKey操作

含义：如果1s内testHotKey这个资源中的第1个参数的次数超过1，就会触发熔断，执行deal_testHotKey操作，但是如果第1个参数值为Cooper，那么阈值就是200

系统规则

相当于是系统入口的大门检测，如果达到一定的条件，整个大门就会关闭

如果整个系统的QPS总和达到10，就会不再处理请求，除非恢复
