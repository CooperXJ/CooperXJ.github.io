BeanDefintion

含义

Spring Framework中定义Bean的配置元信息接口，包含：

  • Bean的类名
  • Bean行为的配置元素，比如作用域、自动绑定模式、生命周期的回调（初始化、回调）等
  • 其他Bean引用
  • 配置设置，比如Bean的属性

BeanDefintion构建方式

  1. BeanDefintionBuilder

  2. AbstractBeanDefinition以及派生类

Bean的名称

  • 指定名称
  • 不指定名称默认会调用命名方法（defaultNameGenerator），其名称为（类型+# +count） （count是从0开始计数的）

初始化Spring Bean

@PostConstruct

实现InitializingBean接口的afterPropertiesSet() 方法

自定义初始化方法

  • XML配置 <bean init-method="init" .../>

  • Java注解 @Bean(initMethod="init")

  • Java API: AbstractBeanDefinition

三者顺序

@PostConstruct UserFactory 初始化中…
InitializingBean initUserFactory(): UserFactory 初始化中…
自定义初始化方法 initUserFactory(): UserFactory 初始化中…

即@PostConstruct -> InitializingBean initUserFactory() -> 自定义初始化方法

延迟初始化

XML 配置：<bean lazy-init=”true” …/>

Java注解 ：@Lazy(true)

延迟加载指的是Spring应用上下文启动之后之后才会进行加载
