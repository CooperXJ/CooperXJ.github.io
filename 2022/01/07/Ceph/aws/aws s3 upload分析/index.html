aws s3 upload分析

发表于 2022-01-07 更新于 2021-08-27 分类于 aws Valine：
本文字数： 11k 阅读时长 ≈ 10 分钟

上传过程

上传可以有多种情况：单个文件上传、多个、文件夹上传，可以参考：

https://docs.amazonaws.cn/sdk-for-java/v1/developer-guide/examples-s3-transfermanager.html#transfermanager-upload-directory

  1. 对上传的对象请求的元数据进行填充

    如果请求中有元数据，那么使用请求的元数据

    如果没有则进行填充

  2. 设置监听器

  3. 提交下载任务并设置monitor

  4. 判断是否可以进行分段上传，如果<=16M，则不进行分段上传，否则进行分段上传

  5. 对每次上传的大小和类型进行一个合适的选择，为分段传输做准备

    • 如果是加密传输，则需要大小和传输进行特殊的选择，比如大小方面需要进行填充，传输需要选择加密传输方式
    • 如果是一般的传输，则长度的选择伪Max（(double)contentLength / (double)MAXIMUM_UPLOAD_PARTS，configuration.getMinimumUploadPartSize()），因为每次分段传输有一个限制就是分的段数不能超过10000（默认，可以修改）

  6. 对是否可以并行传输做一个判断，如果可行则进行并行传输，否则串行传输

    此处需要注意一下，加密的数据不能使用并行传输，因为前后chunk之间有联系

  7. 并行：

    1. 将未上传的任务提交到线程池中，每个任务调用s3提供的底层方法——-PartETag partETag = s3.uploadPart(request).getPartETag();

    2. 该方法每一次上传都会去找已经上传完的part，也就是identifyExistingPartsForResume()

  8. 串行

    串行是将每个chunk一次进行上传

  9. 关于上传校验

    系统中设置如果设置了变量DISABLE_GET_OBJECT_MD5_VALIDATION_PROPERTY，可以System.getProperty(DISABLE_GET_OBJECT_MD5_VALIDATION_PROPERTY)来查看是否已经设置了，如果已经设置了，那么s3会默认禁用上传校验功能。

    默认是会启用校验功能的。但是只要你的DISABLE_GET_OBJECT_MD5_VALIDATION_PROPERTY设置了任何的值，那么就会认为禁用了上传校验功能。

    那么它是如何校验的呢？

    如果提供了MD5 digist的值，那么会和返回的Etag进行比较，如果一致则认为合格，否则会认为该上传块存在问题。
