  2. 用户正常提交给服务器，但是由于网络原因导致用户迟迟未收到服务器的响应，用户继续进行提交操作。

  3. 用户正常提交给服务器，服务器也没有延迟。用户回退浏览器进行重新提交。

    2和3的解决方案：

方法一：

​ 1. 设置状态码

​ resp.setStatus(302)

2. resp.setHeader("Location",xxxxx) (xxxx为具体重定向地址)

方法二：

​ resp.setRedirect(xxxxxx) (xxxx为具体重定向地址)

特点：

- 浏览器地址栏会发生变化
- 会向服务器发送两次请求
- 不共享request域中的资源，因此两次请求request不是同一个request，所以不能共享
- 不能访问WEB-INF下的资源
- 可以访问工程以外的资源

  5. 接口

  1. channel

    • 特点：

      • 是一个队列

      • 线程安全，多个goroutine访问时不需要加锁

      • channel是有类型的

      • channel是引用类型 这就说明了只要将channel传递给多个函数，那么调用的始终都是同一个channel

      • channel必须初始化才能写入数据，即make后才能使用 make(chan int,3)

    • 定义

      var 变量名 chan 类型

      比如 var intChan chan int 即该管道只能写入整数

    • 操作

      • 写入数据

        intChan<- 10

      • 加入的数据不能超过其长度的，如果加入的超过本身的长度会导致死锁

      • 取数据

        Var num int

        num=<-intChan

      • 在没有使用协程的情况下，如果我们的管道数据已经全部取出，再取就会报告死锁

        因为此时后面的代码永远无法执行

      • 管道内的数据可以不接受，取出来之后直接扔掉，也就是<-intChan

      • 如果想在管道内放不同的类型的变量，那么定义到时候需要使用 var allChan chan interface{}，取出来的时候记得使用类型断言

      • 管道关闭

        使用内置的函数close

        当channel关闭后，就不能再向channel写数据了，但是仍然可以从该channel读数据

        close(intChan)

      • 管道遍历

        不要用for循环，因为管道的长度会发生变化

        使用for-range进行遍历,但是需要记得将管道关闭，否则最后会出现死锁

        for v:=range 管道名{

        ​ fmt.Println(“v=”,v)

        }

      • 如果想要存放任意类型数据到管道中，那么可以将管道的类型声明为空接口，最终取出来的时候要对其使用类型断言，但是不能直接取出来用，会报错的

    • 注意事项

      当管道的读写速度不匹配的时候，不会发生死锁，但是只有写没有读便会死锁

docker compose

安装
启动

在有docker-compose.yml的文件目录下启动docker-compose

  • 启动docker-compose up

  • 停止 docker-compse down

  • 重新部署打包 docker-compose up --build

docker-compose 每次启动之后都会自己创建一个网络，用于内部的通信，项目中的所有容器都在同一个网络下，这样的话彼此之间可以通过域名访问

yaml文件规则

docker swarm

初始化节点

docker swarm init

如果这些机器不在同一个内网中，需要彼此之间做认证才行

只有管理者节点才可以执行命令，worker节点无法执行

Raft协议（保证大多数结点存活才可以使用）

比如双主双从，其中还只要有一个主节点挂机，那么另外一个主节点也将不可以用

可以将其他结点从集群中离开，只需要在该离开的结点中输入：docker swarm leave

service

服务可以动态的扩缩容，对于上层的调用者来说他就是一个

docker service

docker service ps 服务名

  • docker service update --replicas {副本数量} {服务名} 或者 docker service scale {服务名}={副本数量}

通过该命令创建的副本会随机分布在集群的主机上，并且通过集群中的每一个节点都可以对服务进行访问，不管该节点有没有该服务

docker swarm 网络

当docker swarm 搭建集群成功之后，会自动生成一个ingress网络，该网络的类型是overlay

overlay可以使得不同docker的内部网络相互ping通，使得集群间的网络成为了一个整体
