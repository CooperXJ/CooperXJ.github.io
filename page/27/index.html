安装Container runtimes

该组件是运行容器使用的，除了docker之外有多种选择

  1. 设置镜像源

    设置docker-ce源 阿里云镜像源

    在/etc/yum.repos.d目录下

    curl -o docker -ce.repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

    下载下来之后将其拷贝到另外两台节点上面去

    (有必要的话基础的源也需要指向国内，就是使用镜像的源，比如centos的源)

  2. 安装docker（每个机器上面都要安装）

    重新docker服务并设置开机生效

    systemctl restart docker && systemctl enable docker

    查看docker服务启动状况

    systemctl status docker

    查看引擎详情(看cgroup driver是否是systemd，不是的需要进行修改)

    docker info

    生成docker配置文件

    重启docker服务

    systemctl restart docker

    将该文件复制到其余节点上,同时重启docker 服务

    scp /etc/docker/daemon.json {node}:/etc/docker/

安装kubeadm（保证机器配置最起码2G内存和2核心）

  1. 将ipatbles和bridged traffic 结合起来 (每台机器都要)

    查看状态

    sysctl -a|grep net.bridge.bridge-nf

  2. 放开对应的端口

    如果方便的话直接将防火墙关闭掉

    systemctl disable firewalld && systemctl stop firewalld

    还有selinux也需要关掉

    1. 安装kubeadm（部署k8s集群）、安装kubelet（管理容器的）、安装kubectl（命令行工具）

      部署源(节点都需要布置)

      安装(每个节点都需要安装)

      yum install kubeadm kubelet kubectl -y

      安装完毕之后强拉kubelet服务的话是不行的，因为环境没有准备好了

      将kubelet服务开机启动,否则下次开机的话无法启动k8s服务

      systemctl enable kubelet

    2. 初始化

      指定pod网络进行初始化(必须要指定16位)

      kubeadm init --pod-network-cidr 192.168.0.0/16 （这里千万要注意，不能与主机所在的网段冲突）

      遇到问题请点击

      会遇到错误，因为swap是不支持的

      需要禁用swap

      1. 安装工具包

        yum install util-linux

      2.  查看swap空间位置

`cat /etc/fstab`

3. 禁用swap空间(每台机器都需要)

`swapoff /dev/mapper/centos-swap`

将/etc/fstab 文件中的swap注释掉，否则下次开机会自动启用swap空间

    执行成功之后需要记录下信息

    可能遇到的问题

    查看当前节点：kbuectl get nodes

    The connection to the server localhost:8080 was refused - did you specify the right host or port?

    原因：kubenetes master没有与本机绑定，集群初始化的时候没有设置

    解决办法： export KUBECONFIG=/etc/kubernetes/admin.conf

  3. 其他节点加入集群

    查看ip_forward是否开启，确保其开启

    输入之前成功初始化的命令

    查看集群情况

    kubectl get nodes

  4. 安装网络插件 （让宿主机之间都联系起来）

    获取yaml文件

    curl https://docs.projectcalico.org/manifests/calico-etcd.yaml -o calico.yaml

    执行

    kubectl apply -f calico.yaml

    查看集群健康状态

    kubectl get ds n kube-system

    kubectl get pods -o wide -n kube-system

    kubectl get cs

  5. 命令补全

    安装包

    yum install bash-completion

    source /usr/share/bash-completion/bash_completion

    配置包

    kubectl completion bash >/etc/profile.d/kubectl.sh

    加载配置文件

    source /etc/profile.d/kubectl.sh

    vim /root/.bashrc 在里面添加上面这句话，这样的话每次开机就可以生效

    完成！

Helm的好处

  1. 使用helm可以将yaml作为一个整体管理
  2. 实现yaml的高效复用
  3. 使用helm应用级别的版本管理

Helm的概念

  • helm

    命令行客户端工具

  • Chart

    把yaml打包，是yaml集合

  • Release

    基于chart部署实体，应用级别的版本管理

版本架构

Helm命令

  • 添加存储库

    helm repo add {name} {地址}

  • 更新仓库

    helm repo update

  • 查看配置的存储库

    helm repo list

  • 删除存储库

    helm repo remove {name}

  • 搜索应用

    helm search repo {app name}

  • 安装

    helm install {自定义应用名称} {搜索应用找到的app}

  • 查看安装的应用

    helm list

  • 查看应用状态

    helm status {name}

Chart相关

  • 创建chart

    helm create chart {name}

    生成的文件夹下由如下文件：

    • Chart.yaml

      当前chart属性配置信息

    • templates

      编写yaml文件存放到这个目录下

    • values.yaml

      yaml文件可以使用全局变量

  • 安装chart

    helm install {name} {自定义的chart文件夹，也就是上一步创建的yaml文件夹}

  • 更新

    helm update {name} {自定义的chart文件夹，也就是上一步创建的yaml文件夹}

  • 高效复用yaml文件

    通过传递参数，动态渲染模板，yaml内容动态传入参数生成

    在chart中有values.yaml文件，定义yaml文件全局变量

    yaml文件大体主要有以下几个方面不同：

    • image
    • tag
    • label
    • port
    • replicas

    流程：

    1. 在values.yaml中定义变量和值

    2. 在具体的yaml文件获取定义的变量值

      • 通过表达式使用全局变量

        {{ .Values.变量名称 }}

      • 应用发布编号（自动生成）

        {{ .Release.Name }}

k8s安全机制

发表于 2022-01-07 更新于 2022-01-17 分类于 K8S Valine：
本文字数： 462 阅读时长 ≈ 1 分钟

概述

  1. 访问k8s集群的时候，需要经过三个步骤完成具体的操作

    1. 认证
    2. 鉴权（授权）
    3. 准入控制

  2. 在进行访问的时候，过程中都需要经过apiServer，apiServer做统一协调，就像门卫。

    访问过程中需要证书，token，或者有用户名+密码，如果访问Pod，需要serviceAccount

详细过程

  1. 认证

    • 传输安全

      对外不暴露8080端口，只能内部访问，对外使用端口6443

    • 认证

      客户端身份认证常用方式：

      • https 证书认证，基于CA证书
      • http token认证，通过token识别用户
      • http 基本认证，通过用户名+密码识别用户

  2. 鉴权（授权）

    基于RBAC进行鉴权操作（基于角色访问控制）

    过程：

    1. 创建命名空间

    2. 在新的命名空间中创建pod

    3. 创建角色

    4. 查看角色

      kubectl get role -n {namespace}

    5. 创建角色绑定

    6. 查看角色绑定

      kubectl get role,rolebinding -n {namespace}

    7. 使用证书识别身份

      这个需要上网自己查找对应的格式

  3. 准入控制

    就是准入控制器的列表，如果列表有请求内容则通过，没有就拒绝

yam文件组成以及各个字段含义

  1. 组成

    • 控制器定义
    • 被控制对象

  2. yaml文件各个字段含义

​ 对于containers中的command和args分别相当于是docker中的entrypoint和cmd

​

​ 具体策略如下：

  1. 如果没有提供command和args，那么会使用docker镜像默认的

  2. 如果提供了command但是没有提供args，那么生效的是command，docker images默认的cmd和entrypoint会被忽略

  3. 如果只提供了args，docker image默认的entrypoint会执行你提供的args，默认的args会被忽略

  4. 如果你提供了command和args，默认的entrypoint和cmd会被忽略，你的cmd和args会生效

快速生成yaml文件

  1. 当应用还没有生成的时候

    kubectl create {资源类型，比如deployment} {name} --image={iamge name} -o yaml --dry-run > 生成yaml位置

    其中–dry-run一定要加上

  2. 当应用已经生成了，导出他的yaml文件

    kubectl get {资源类型，比如deploy} {资源名称} -o=yaml --export > yaml文件位置

Secret

定义

Secret也是一种资源形式

作用：加密数据存在etcd中，让Pod容器以挂载Volume方式进行访问

场景：凭证

使用

  • yaml文件（值都需要使用base64进行编码）

  • 挂载（下面的mysecret都是利用yaml文件创建的Secret资源的name）

    • 以变量形式挂载到Pod容器中

    • 以Volume形式挂载到Pod容器中

ConfigMap

作用

存储不加密的数据到etcd，让Pod以变量或者Volume挂载到容器中

场景：配置文件

使用

  • 创建配置文件

    就是平常写的properties、json、yaml文件

  • 创建configMap

    kubectl create configmap {configmap name} --from-file={配置文件}

    kubectl get cm

    kubectl describe cm {configmap name}

  • 挂载和Secret类似

    • 变量形式挂载

    • Volume形式挂载

蓝绿发布

对于相同的资源yaml文件复制一份，然后将按照需要进行修改，最后将前一个资源对应的 Service中的label改为指向修改过后的资源即可。

滚动升级

原理：逐个进行替换

在资源文件中设置相应的策略

金丝雀发布

原理：多版本共存,就是先派一个进行测试，没有问题的话就进行完全发布更新

操作：

  1. 手动更新镜像并停止滚动发布

    kubectl set image {资源类型} {name} {镜像} = {镜像版本} && kubectl rollout pause {资源类型} {name}

  2. 如果发现没有问题则完全发布

    kubectl rollout resume {资源类型} {name} &&kubectl rollout pause {资源类型} {name}
